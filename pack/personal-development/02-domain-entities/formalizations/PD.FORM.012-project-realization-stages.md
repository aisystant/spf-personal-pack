# Формализация: Стадии реализации проектов (Project Realization Stages)

## 1. Определение

**Стадии реализации проектов** — формализация пятистадийной цепочки, через которую проходит любой проект от замысла до результата. Каждая стадия имеет свой метод, входы и выходы.

## 2. Пять стадий

| # | Стадия | Метод | Входы | Выходы |
|---|--------|-------|-------|--------|
| 1 | **Целеполагание через стратегирование** (Goal-setting via Strategizing) | PD.METHOD.008 | Неудовлетворённости, заметки | Список неудовлетворённостей, стратегия, проекты |
| 2 | **Моделирование системы** (System Modeling) | — | Описание проекта | Описание системы, список задач с бюджетом |
| 3 | **Формулирование работ через планирование** (Work Formulation via Planning) | PD.METHOD.009 | Задачи с бюджетом | План работ, календарь |
| 4 | **Исполнение через прикладное мастерство** (Execution via Applied Mastery) | Прикладные методы | План работ | Рабочие продукты |
| 5 | **Анализ обратной связи** (Feedback Analysis) | PD.METHOD.008 | Результаты, обратная связь | Новые гипотезы → возврат к стадии 1 |

## 3. Цепочка

```
[1] Целеполагание (стратегирование)
    ↓ проекты
[2] Моделирование системы
    ↓ задачи с бюджетом
[3] Формулирование работ (планирование)
    ↓ план работ
[4] Исполнение (прикладное мастерство)
    ↓ рабочие продукты
[5] Анализ обратной связи
    ↓ новые гипотезы → [1]
```

## 4. Роль беглых заметок

> Беглые заметки (fleeting notes) питают **каждую** стадию, а не только стадию 1. На любой стадии могут возникнуть наблюдения, идеи и неудовлетворённости, которые фиксируются как беглые заметки и обрабатываются на сессии стратегирования.

## 5. Типичные ошибки

| Ошибка | Стадия | Последствие |
|--------|--------|-------------|
| Пропуск моделирования | 2 | Задачи без бюджета, невозможно планировать |
| Пропуск формулирования работ | 3 | Прокрастинация из-за неясности "что именно делать" |
| Исполнение без целеполагания | 4 | Работа ради работы, нет связи со стратегией |
