# Bounded Context: Характеристики и состояния созидателя

---

## Domain Name

**Характеристики и состояния созидателя**

(English: Characteristics and States of Creator)

---

## Object of Description

**Созидатель** как объект инженерного описания.

| Созидатель IS | Созидатель IS NOT |
|---------------|-------------------|
| Объект описания с характеристиками | Конкретный человек |
| Носитель измеримых атрибутов | Субъект обучения |
| Экземпляр типа (как автомобиль, здание) | Личность с биографией |

Созидатель описывается **аналогично инженерному объекту**: через характеристики, состояния, методы испытаний и рабочие продукты оценки.

---

## Scope (Что входит)

| Элемент | Описание | Пример |
|---------|----------|--------|
| **Характеристики** | Стабильные атрибуты созидателя | Агентность, калибр личности |
| **Состояния** | Временные режимы функционирования | Состояния ученика |
| **Роли** | Позиции в деятельности (как режимы участия) | Агент, аналитик, ментор |
| **Индикаторы** | Наблюдаемые признаки характеристик/состояний | Количественные и качественные |
| **Методы оценки/испытаний** | Способы определения характеристик | Assessment methods |
| **Рабочие продукты оценки** | Верифицируемые результаты оценки | Assessment results |
| **Failure modes** | Типичные ошибки интерпретации | Confusion patterns |
| **SoTA** | Статус актуальности утверждений | current/hypothesis/deprecated |

---

## Non-Goals (Что НЕ входит)

| Исключено | Причина | Где должно быть |
|-----------|---------|-----------------|
| Обучение как курсы, уроки, программы | Дидактика — downstream | Learning systems |
| Маршруты развития | Траектории — downstream | Development programs |
| Сценарии переходов между состояниями | Сценарии — downstream | Guides, coaching |
| Терапия, клиника | Другой домен | Mental health pack |
| AI-индексы как источник истины | Embeddings ≠ knowledge | AI implementations |
| «30-дневные программы» | Дидактика | Learning platforms |
| Мотивационный контент | Downstream engagement | Content marketing |

---

## Truth Criteria

Утверждения в этом pack'е считаются обоснованными, если:

| Критерий | Тест |
|----------|------|
| **Наблюдаемые индикаторы** | Характеристика имеет специфицированные индикаторы, которые можно наблюдать |
| **Проверяемые рабочие продукты** | Метод оценки производит верифицируемый артефакт |
| **Distinction test** | Каждое различение имеет тест «как отличить X от Y» |
| **SoTA + revision criterion** | Каждый статус имеет условие пересмотра |

**Запрещено**:
- Утверждения без наблюдаемых индикаторов
- Методы без рабочих продуктов
- SoTA без revision criterion

---

## Scope Clarification: Learning

**Обучение как процесс** (Learning as process of agent change) — входит в scope этого Pack'а. Это мета-концепт о том, КАК описания трансформируются в capabilities через практику и обратную связь. См. [PD.FORM.004](../02-domain-entities/formalizations/PD.FORM.004-agent-cognitive-layers.md).

**Обучение как курсы/программы** (Learning as curricula) — НЕ входит в scope. Это downstream-реализация.

| В scope | НЕ в scope |
|---------|-----------|
| Обучение как процесс изменения агента | Курсы, уроки, программы |
| Онтология: описание → знание → навык | Методики преподавания |
| Failure modes перехода | Учебные материалы |

---

## Downstream Note

**Траектории развития** — вне scope этого pack'а.

| Pack описывает | Downstream использует |
|----------------|----------------------|
| Что такое агентность | Как развить агентность |
| Какие состояния существуют | Как перейти между состояниями |
| Методы оценки | Программы оценки и развития |
| Индикаторы | Критерии прогресса |

Pack — **source-of-truth** (что существует).
Downstream — **implementation** (как использовать).

## Связь с FPF

Этот pack опирается на FPF-различения:

- **Объект / описание / носитель**: Созидатель (объект) ≠ карточка характеристики (описание)
- **Метод / сценарий**: Методы оценки — не пошаговые инструкции
- **Знание / представление**: Pack — знание; downstream — представления
