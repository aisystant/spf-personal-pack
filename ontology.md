# Онтология: Характеристики и состояния созидателя

> Онтология домена по SPF.SPEC.002.
> Полный реестр типов сущностей, ключевых понятий и связей для Pack PD.
> Каждое понятие привязано к родительскому понятию из базовой онтологии SPF (SPF.SPEC.002, секция 2).

---

## 1. Виды сущностей

> Все типы, используемые в этом Pack: базовые (из SPF) + расширенные (определённые этим Pack).
> **Обязательная колонка «FPF-понятие»** — родительское понятие из базовой онтологии SPF.

### 1.1. Базовые виды (из SPF)

| Код | Вид | FPF-понятие | Определение | ≠ (что это НЕ) |
|-----|-----|-------------|-------------|-----------------|
| `M` | Метод | U.Method | Описание способности (capability) производить рабочий продукт | ≠ инструмент (D.001), ≠ навык (D.007) |
| `WP` | Рабочий продукт | U.Work + U.Episteme | Наблюдаемый результат применения метода; существует на носителе | ≠ описание (D.005) |
| `FAIL` | Режим ошибки | — (SPF-специфичный) | Типичное нарушение интерпретации с наблюдаемыми симптомами | ≠ ошибка человека, ≠ мнение |
| `D` | Различение | A.7 Strict Distinction | Концептуальная граница, нарушение которой создаёт неустранимую путаницу | ≠ факт, ≠ определение |
| `MAP` | Карта | U.Episteme | Навигационная структура связей между сущностями | ≠ контент |
| `CHR` | Характеристика | U.Characteristic | Измеримая ось оценки созидателя | ≠ индикатор (наблюдаемый признак) |
| `SOTA` | SoTA-аннотация | — (SPF-специфичный) | Статус актуальности утверждения + критерий пересмотра | ≠ обзор литературы |

### 1.2. Расширенные виды (определены этим Pack)

| Код | Вид | FPF-понятие | Определение | ≠ (что это НЕ) |
|-----|-----|-------------|-------------|-----------------|
| `ARCH` | Архитектура | U.System + U.Episteme | Архитектурное описание модели созидателя: слои, уровни | ≠ реализация, ≠ методология |
| `ROLE` | Роль | U.RoleAssignment | Контекстная функция созидателя в деятельности | ≠ должность, ≠ человек (D.002) |
| `STATE` | Состояние | U.Flow | Временный режим функционирования созидателя | ≠ характеристика (стабильное), ≠ этап обучения |
| `FORM` | Формализация | U.Episteme | Концептуальное описание структуры (программа, направление, уровни) | ≠ план действий, ≠ курс |
| `QUAL` | Квалификация | U.Kind | Классификационная единица уровня подготовки | ≠ оценка, ≠ сертификат |

## 2. Глоссарий домена

> Ключевые понятия предметной области. Для полных определений — см. [01A-bounded-context.md](01A-bounded-context.md).
> **Обязательная колонка «Родительское понятие (SPF)»** — к какому универсальному понятию из базовой онтологии SPF относится этот доменный термин.

| Термин | Определение | Родительское понятие (SPF) | Связан с |
|--------|-------------|---------------------------|----------|
| **Созидатель** | Объект описания с характеристиками — носитель измеряемых атрибутов (агентность, масштаб личности). Не конкретный человек, а инженерный объект | U.System | PD.ARCH.001 |
| **Характеристика созидателя** | Стабильный измеряемый атрибут созидателя (агентность, масштаб личности, собранность) | U.Characteristic | PD.CHR.001 |
| **Состояние** | Временный режим функционирования созидателя; определяется текущим набором характеристик | U.Flow | PD.STATE.001 |
| **Роль в деятельности** | Функциональная позиция созидателя: агент, аналитик, наставник, архитектор. Один человек может исполнять разные роли | U.RoleAssignment | PD.ROLE.001–005 |
| **Знание** | Способность агента надёжно достигать результатов. Не информация, не описание, не навык | U.Capability | PD.D.013, PD.D.014 |
| **Навык** | Процедурный слой знания — автоматизированная последовательность действий. Часть знания, не весь объём | U.Capability | PD.D.007, PD.D.014 |
| **Мировоззрение** | Метафреймворк интерпретации агента: онтология, ценности, методы. Живёт в агенте, не в репозитории | U.Episteme | PD.D.015 |
| **Учёт времени** | Метод регистрации потраченного времени для анализа инвестирования внимания | U.Method | PD.METHOD.001 |
| **Индикатор** | Наблюдаемый/вычисляемый признак состояния созидателя (количественный или качественный) | U.Characteristic | PD.CHR.001 |
| **Формализация** | Концептуальная структура для описания программ, направлений, уровней зрелости | U.Episteme | PD.FORM.001–004 |

## 3. Отношения между видами

| Субъект | Отношение | Объект | Пример |
|---------|-----------|--------|--------|
| Метод (M) | порождает → | Рабочий продукт (WP) | PD.METHOD.001 → PD.WP.001 |
| Failure Mode (FAIL) | нарушает ← | Метод (M) | PD.FAIL.001 ← PD.METHOD.001 |
| Характеристика (CHR) | описывает → | Роль (ROLE) | PD.CHR.001 → PD.ROLE.001 |
| Состояние (STATE) | определяется ← | Характеристика (CHR) | PD.STATE.001 ← PD.CHR.001 |
| Формализация (FORM) | структурирует → | Роль (ROLE) | PD.FORM.001 → PD.ROLE.001 |
| Архитектура (ARCH) | содержит → | Роль (ROLE) | PD.ARCH.001 → PD.ROLE.* |
| Архитектура (ARCH) | содержит → | Характеристика (CHR) | PD.ARCH.001 → PD.CHR.001 |

## 4. Иерархия типов

```
Созидатель (PD.ARCH.001) — объект описания
├── Характеристики (CHR) — стабильные атрибуты
│   ├── Агентность
│   ├── Масштаб личности
│   └── Собранность
├── Состояния (STATE) — временные режимы
├── Роли (ROLE) — позиции в деятельности
│   ├── Ученик (PD.ROLE.001)
│   ├── Интеллектуал (PD.ROLE.002)
│   ├── Профессионал (PD.ROLE.003)
│   ├── Исследователь (PD.ROLE.004)
│   └── Просветитель (PD.ROLE.005)
└── Формализации (FORM) — структуры описания
    ├── Программы развития (PD.FORM.001)
    ├── Направления развития (PD.FORM.002)
    ├── Зрелость ученика (PD.FORM.003)
    └── Когнитивные слои (PD.FORM.004)
```
